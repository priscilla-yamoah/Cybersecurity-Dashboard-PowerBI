# 🛡️ Power BI Cybersecurity Threat Analysis Dashboard

## Project Overview
This project features a **3-page Power BI dashboard** built for **actionable intelligence** on network security threats. The primary goal is to shift focus from just reporting "what happened" to identifying **system failures** and providing the **raw data** needed for immediate threat mitigation.

The report provides a comprehensive flow of analysis: **Overview → System Weaknesses → Actionable Detail.**

---

## 🔑 Key Features & Actionable Insights
This dashboard enables security teams to move swiftly from insight to action:

1.  **System Failure Quantification:** Quantifies the percentage of high-severity attacks that were **'Ignored'** or only **'Logged'** (Page 2), highlighting critical blind spots.
2.  **Weak Link Mapping:** Identifies the **Top Attacking Cities**, most exploited **Protocols (TCP/UDP)**, and **Traffic Types** to guide firewall policy updates.
3.  **Actionable Drill-Through:** Developed a **Drill-Through page (Page 3)** providing specific **Source IP Addresses** and **Timestamps** for quick security response and IP blocking.
4.  **Trend Analysis:** Visualizes attack trends over time and geographical hotspots (Page 1).

---

## 🛠️ Tools and Techniques Used
* **Data Visualization:** Microsoft Power BI Desktop
* **Data Analysis:** DAX (Data Analysis Expressions) for complex metrics.
* **Advanced Filtering:** **Drill-Through** feature, **Top N filtering**, and advanced slicer application for interactive analysis.
* **Design:** Custom dark theme for professional presentation and clear information hierarchy.

---

## 📊 Report Pages
| Page | Focus Area | Goal |
| :--- | :--- | :--- |
| **Page 1** | **Threat Overview** | High-level summary, KPIs, and overall attack trends. |
| **Page 2** | **Weaknesses & Failures** | Deep dive into system efficacy and vulnerability analysis. |
| **Page 3** | **Attack Detail & Source Data** | Raw, actionable log data for quick threat mitigation. |

## ⬇️ How to View the Report
